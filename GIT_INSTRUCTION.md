# Инструкция по отправке изменений в GitHub

## Быстрый способ (одной командой)

Используйте созданный скрипт:

**PowerShell:**
```powershell
.\git-push.ps1 "Ваше сообщение коммита"
```

**Batch (CMD):**
```cmd
git-push.bat "Ваше сообщение коммита"
```

---

## Пошаговая инструкция (вручную)

### Шаг 1: Инициализация репозитория (если еще не сделано)

```bash
git init
```

### Шаг 2: Добавление удаленного репозитория (если еще не добавлен)

Если у вас еще нет репозитория на GitHub:
1. Зайдите на https://github.com
2. Создайте новый репозиторий (New repository)
3. Скопируйте URL репозитория (например: `https://github.com/username/repo-name.git`)

Затем выполните:
```bash
git remote add origin https://github.com/ваш-username/ваш-репозиторий.git
```

Если репозиторий уже добавлен, проверьте:
```bash
git remote -v
```

### Шаг 3: Добавление файлов в индекс

```bash
git add .
```
(добавляет все файлы) или
```bash
git add имя_файла
```
(добавляет конкретный файл)

### Шаг 4: Создание коммита

```bash
git commit -m "Описание изменений"
```

Например:
```bash
git commit -m "Обновлен дизайн главной страницы"
```

### Шаг 5: Отправка в GitHub

**Первый раз:**
```bash
git push -u origin main
```
или (если ветка называется master):
```bash
git push -u origin master
```

**В последующие разы:**
```bash
git push
```

---

## Полный пример (первый раз)

```bash
# 1. Инициализация
git init

# 2. Добавление удаленного репозитория
git remote add origin https://github.com/ваш-username/ваш-репозиторий.git

# 3. Добавление всех файлов
git add .

# 4. Создание коммита
git commit -m "Первая загрузка проекта"

# 5. Отправка в GitHub
git push -u origin main
```

---

## Полный пример (последующие изменения)

```bash
# 1. Добавление изменений
git add .

# 2. Создание коммита
git commit -m "Описание изменений"

# 3. Отправка в GitHub
git push
```

## Полезные команды

**Проверить статус:**
```bash
git status
```

**Посмотреть историю коммитов:**
```bash
git log
```

**Проверить удаленные репозитории:**
```bash
git remote -v
```

**Просмотреть изменения:**
```bash
git diff
```

## Решение проблем

### Ошибка: "fatal: not a git repository"
Решение: Выполните `git init` в корневой папке проекта

### Ошибка: "remote origin already exists"
Решение: Репозиторий уже добавлен, просто используйте `git push`

### Ошибка: "Authentication failed"
Решение: Нужно настроить авторизацию GitHub:
- Личный токен доступа (Personal Access Token)
- Или SSH ключи

### Ошибка: "branch 'main' has no upstream branch"
Решение: Используйте `git push -u origin main` первый раз
